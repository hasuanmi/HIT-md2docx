# 学位层次适配（步骤 7）

在已生成的**硕士格式** docx 上，按步骤 0 确认的 `DEGREE` 做封面临门一脚：把封面 / 页眉 / 页脚 / 后置小节标题里的「硕士 / master」改写成目标层次对应词。

## 为什么单独成步（而非改引擎 / profile）

- 引擎只内置 `hit-master-thesis` 一套规范。本科/博士的版式差异（封面用词、页眉用词）本质是**文本替换**，不需要另写 profile，也不该侵入引擎（违反铁律 #1）。
- 故此作为生成后的 docx 后处理步骤固化：`degree_adapt.py`。`DEGREE=master` 时为空操作，硕士论文照常。

## 步骤

在步骤 6（report）之后执行（封面已在步骤 3.5 注入，故放在最后改词最稳）：

```bash
python skill/scripts/degree_adapt.py output/<论文>.docx \
    --degree <bachelor|master|doctor> \
    --out output/<论文>.docx
```

- `--degree bachelor` → 封面/页眉「硕士」→「本科」、「master」→「bachelor」
- `--degree doctor`  → 封面/页眉「硕士」→「博士」、「master」→「doctor」
- `--degree master`  → 空操作，原样保留（默认）

## 该脚本做了什么

1. **docx 包内改写**：直接读 `word/document.xml` 与各 `word/header*.xml` / `word/footer*.xml` 的 XML，找所有 `<w:t>` 文本节点。
2. **块内逻辑重组 + 位置 preserving 重排**：封面模板常把「硕士」拆成多个跑（如「硕」+「士研究生」），逐跑替换会漏。脚本在每个块（段落/表格）内拼接逻辑文本、统一替换、再按原字形写回——既修正拆分，又尽量保留原字体/加粗。
3. **作用范围**：
   - 封面区（从文档开头到首个一级标题 / 摘要 / 声明之前）：全量改写，覆盖中文封面「硕士学位论文」「硕士研究生：」、英文封面「Master Degree」等。
   - 页眉 / 页脚：整部件改写。
   - 正文后置小节标题「攻读硕士学位期间取得的科研成果」：整体一致改写。
   - 正文一般内容（作者偶然写到「硕士」）不动，避免误伤。

## 健壮性

- **幂等**：对已适配文档再跑同 `DEGREE` 不会二次变化（本科→本科 不变）；master 空操作。
- **稳健改写**：XML 级处理，普通段落 / 表格单元格 / 文本框中的学位词均能命中。
- **不改引擎 / profile**：符合铁律 #1，不会引入生成器回归。

## 注意

- 若重跑引擎（步骤 3）或重新注入封面（步骤 3.5），硕士用词会被还原，需再跑一次本步骤恢复目标层次。
- 仅替换学位语境词；作者正文、专业术语中的「硕士」如确需保留，不会被误改（正文区已排除）。
