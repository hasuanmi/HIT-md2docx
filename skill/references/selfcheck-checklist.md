> 改规则只改 `format_rules.py`（单一真相来源），本文件同步更新即可，不必改生成代码。
> 完整审查要点见 `hit-master-review-points.md`（往届易错点速查 10 条）。

## 执行方式

```bash
# 只读审计，输出 JSON
python skill/scripts/audit_docx.py thesis.docx --json audit.json
# 自动修复可修复项（幂等），随后重新审计验证
python skill/scripts/fix_docx.py thesis.docx --out thesis.fixed.docx
# 生成可读问题清单
python skill/scripts/report.py audit.json --md 问题清单.md
```

## 设计原则（防止"改一个 bug 出另一个"）

- **解耦**：校验/修复只作用于"依赖作者 Markdown 内容、生成器保证不了"的项；字体/字号/行距/页眉双线由 `hit_master_thesis` profile 正确生成，不重复校验。
- **数据驱动**：规则集中在 `format_rules.py`，新增/微调规则不碰生成代码。
- **幂等**：自动修复脚本连续运行两次，第二次改动数为 0（已手测验证），避免引入新错。
- **仅检测项**（R03/R04/R07/R08/R09/R10/R11）交人工在 Word/WPS 复核，避免自动修复引入新错。
